## 0. 一句話定位

> **動作技能課練的是「手」（手眼協調、肌肉記憶），不是「腦」（理解判斷）。**
> 「通過」＝動作做得順，不是想通了問題。設計重心因此完全不同。

| 面向 | 概念課（高年級） | 動作技能課（低年級） |
|---|---|---|
| 主要挑戰 | 理解、判斷、對應 | 手眼協調、肌肉記憶 |
| 「通過」定義 | 想通了 | 動作做順了 |
| 失敗處理 | 錯了鎖定＋提示 | **失誤靜音重試，不打斷節奏** |
| 進度感 | 一步步解鎖 | 反覆累積次數 |
| 反饋強度 | 中（避免干擾思考） | **高**（每個動作都要回饋，形成節奏） |
| 文字量 | 精簡但關鍵 | **極端精簡，圖示為主（≤5 字）** |

## 1. 動作技能課的五條核心原則（M1–M5）

### M1 ｜ 練「動作品質」，不是「動作次數」
系統要能分辨**真動作**與**敷衍動作**，否則練習等於白練。
- 移動類：只算「跨螢幕的長距離移動」（例 ≥400px），短距離微調不算。
- 路徑類：必須沿線一定容錯內移動，否則靜音重置這一條。
- 每個動作設**隱形時長下限**（例：一條路徑 ≥3 秒），防止瘋狂拉滑鼠灌次數。

### M2 ｜ 失誤靜音，節奏優先（**這個年齡最重要**）
小一玻璃心，紅框否定會打斷手感練習。
- ❌ 不要：紅色 X、alert、否定文字、「錯 N 次淘汰」。
- ✅ 要：點偏了不彈框不出聲，目標維持原狀等下次；走偏了線條淡出、回起點重來、**不扣任何進度**。

### M3 ｜ 進度視覺化，讓「30 次」感覺可達
30 次一模一樣的動作對小一很漫長，要切成看得見的小段落。
- 例：30 次＝6 朵花 ×5 次填滿一朵；32 次＝8 圈、中央大字顯示第幾圈。
- 每次成功＝一顆小星星飛入進度區（把抽象次數變成具體收集）。

### M4 ｜ 判定區放大，容錯優先
小一游標飄、抓不穩，**判定區要比視覺大很多**。
- 點擊：視覺半徑 40px → 判定 **≥80–100px**。
- 移動目標：視覺 60px → 判定 **120px**。
- 路徑：視覺寬 8px → 容錯 **±40px**。

### M5 ｜ 反饋是「節奏」，不是「說明」
每個成功動作配一小段音效（100–200ms）＋視覺回饋，累積成節奏感。**文字回饋越少越好**。

## 2. 防挫折 vs 反硬 A：兩張不同的表（別混在一起）

低年級要**先**定義防挫折的全域不變式，**再**談防作弊。

### 防挫折（全域不變式，先寫死）
- **絕不卡死**：任何關卡都有保底出口（逾時自動放行、連錯 N 次靜音放行）。
- **失敗非致命**：血量/生命歸零不 game over，補滿再來（實戰驗證有效）。
- **單調可完成**：主線進度只增不減；玩家永遠有一條打得完的路。

### 反硬 A（防假動作，動作技能版的作弊擋法）
| 硬 A 手法 | 擋法 |
|---|---|
| 猛點同一點灌次數 | 兩次動作間插入必要過場（≥1 秒） |
| 小範圍搖游標騙圈 | 每次移動必須跨 ≥400px |
| 亂點所有目標 | 只計「點對」，點錯不算也不扣 |
| 直接拉直線騙路徑 | 沿線 ±40px 才算＋時長 ≥3 秒 |
| 不看畫面猛點 | 完整流程需 ≥2 秒過場、總時長下限 |

> 關鍵區別：**反硬 A 擋的是「假裝有做」；防挫折保的是「真的做不好也不會被懲罰」。** 兩者都要，但語義不同、各一張表。

## 3. Hook / Repeat / Deliver（承諾 → 心跳 → 兌現）

- **Hook（前 3 秒）**：用一個大按鈕＋一句話承諾一個目的，第一次操作本身就是第 1 次練習（無縫承接）。
- **Repeat（過程）**：反覆出現同一句核心訊息（例「我可以用滑鼠讓電腦聽我的話」），每個里程碑都重述一次。
- **Deliver（結尾）**：撒花＋徽章＋金幣三合一 + 明確的**下一步行動**（例「舉手告訴老師」、翻轉教育的「我是小老師」）。Deliver 是兌現 Hook 的承諾，不能省。

## 4. 給小一的硬約束（工程檢查清單）

- [ ] 所有點擊判定半徑 **≥100px**（M4）。
- [ ] **禁**紅色否定 / alert / 「錯 N 次淘汰」（M2）。
- [ ] **禁**倒數計時（破壞手感練習）。
- [ ] 站間文字 **≤5 字**，以大圖示＋箭頭為主。
- [ ] 每個成功動作 **100–200ms 短音效**，累積成節奏（M5）。
- [ ] **游標呼叫**：3 秒沒移動就用漣漪動畫指出游標位置（小一常「找不到箭頭」）。
- [ ] **重來友善**：任何時候可按大 [重來]，不扣進度。
- [ ] 素材一律 AI 生圖，**風格統一（扁平貼紙／深色描邊）＋透明背景＋自動裁切**，且**未載入要能退回向量**（絕不因圖沒到就開天窗）。
- [ ] 存檔續玩 / 帳號金幣 / 資安，**第一版就當一級需求**（別後補，會反覆改 schema）。

## 5. 規格治理：避免「無人察覺的大偏離」

P1U-W01 的教訓：書面 spec 與交付成品在主題、技能、站別上幾乎變成兩堂不同的課，且長期無人發現。防範方法：

1. **正典聲明＋追溯表**：spec 開頭寫明綁定哪個實作檔；任何偏離要回填 spec（change control）。「要改方向，先改 spec。」
2. **可數名詞/單位表**：站、關、回合、次、圈全部標準化，杜絕「10 關」被解讀成回合或站的歧義。
3. **明列 In / Out of Scope ＋跨週技能地圖**：本週只教 X；其餘技能屬 W0X，不在本週。單週 spec 才不會各自把別週的東西吸過來、造成首週範圍過載。
4. **DoD ＋參考範例對抗定性形容詞**：把「像專業遊戲／情緒價值給滿」翻成**可勾選條目**＋一張參考畫面。定性詞一定要配可驗收定義。
5. **互動詞彙表當單一真相**：一次定義「右鍵＝橘色＝『點右鍵』」「滾輪往前＝上升」「雙擊＝×2＝藍色」。可杜絕左右鍵文案不對稱、滾輪方向不一致這類一致性 bug。
6. **認知/識字負荷預算**：小一多在識字初期。標明每畫面最多幾字、是否需識字；若引入閱讀理解，標注前置識字假設。
7. **時長預算表**：每站分鐘 × 次數加總 = 課堂時間，避免內容超時。
