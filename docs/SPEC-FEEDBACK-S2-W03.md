# 給 SPEC 團隊的回饋 — 從 S2-W03《消失的慶典皇冠》學到的

> 這一版做下來，發現幾個「如果 spec 一開始就寫清楚，就能少走很多冤枉路」的地方。
> 以下每一條都是**未來寫 spec 時建議直接納入的規範**，附上這次的實際例子。

---

## 1. 每一段文字，後面都要指定一個「互動確認」

**這次的問題**：spec 通常只給劇情、對話、旁白的文字，沒有規定「玩家讀完要做什麼」。結果玩家可以一路按「繼續」按過去，什麼都沒記住。

**建議 spec 這樣寫**：每一段旁白／對話／揭曉文字，都要在旁邊註明「讀完接什麼確認互動」（例如：把這段的關鍵字拖回句子、選出重點、判斷對錯）。
> 原則：**沒有純粹「按過去」的文字。每段話都要有一個小小的動作證明玩家讀懂了。**

## 2. 「要教的概念」要寫成玩家看得見的動態變化，不是一句教學目標

**這次的問題**：spec 寫「教問題要問多長」，但沒說這個概念長什麼樣子、怎麼玩。

**建議 spec 這樣寫**：直接描述概念的**視覺化互動**。
> 例：太長的話 →畫面上文字被「淹沒」看不到重點；太短的話 →畫面「缺一角」。玩家要把它調到剛好。
> 概念不是用文字解釋，是變成一個玩家能親手操作、看得到變化的東西。

## 3. 謎題邏輯要經得起推敲，而且多張資訊表不能互相矛盾

**這次的問題（最重要）**：原 spec 的「排時間軸」是把每個嫌疑人排在**不同時間**（阿福 8:00、林主席 8:45）。但這在邏輯上不成立——**不在場證明應該是「同一個案發時間，每個人各自在哪、誰能證明」**，不是不同時間各站一格。而且同一關還有另一張「巡邏紀錄表」跟它打架（一張說那時段沒人、一張說大家都在）。

**建議 spec 這樣寫**：
- 出任何推理謎題時，**附一條「正解推理鏈」**：玩家從哪些線索、怎麼一步步推到答案。寫得出來，邏輯才成立。
- 如果一關有**多張資訊表/線索**，要標明每張表**各自回答什麼問題**，並確認它們**不會互相矛盾**。
> 檢查法：找一個聰明的小孩認真想這題，他推得通嗎？兩張表擺一起會不會打架？

## 4. 角色要在「成為謎題核心」之前先登場

**這次的問題**：真兇「林主席」直到解謎那一關才第一次以名字出現，玩家根本不認識她，就被要求把她排進謎題。

**建議 spec 這樣寫**：為每個角色標註「**第一次登場**」和「**在謎題中被使用**」的先後，確保**先鋪陳、後使用**。關鍵角色至少要在前面出現過 2–3 次（哪怕只是被提到名字），登場才自然。

## 5. 需要玩家推理的關卡，要先把「已知資訊」攤給他，不要設計成瞎猜

**這次的問題**：排列/比對的關卡，線索是「排上去之後」才顯示，等於逼玩家試錯。

**建議 spec 這樣寫**：凡是要玩家排列、配對、比對的關卡，spec 要列出玩家**動手前手上已有哪些資訊**（例如四個人各自的供詞先攤開），讓玩家是「用讀到的資訊推理」，不是猜。

## 6. 兩個產品層級的硬規定，請直接寫進 spec

- **進度與金幣一定綁帳號**：換裝置、共用電腦都不會遺失或串號。
- **不提供「跳過」**：靠上面第 1 條的互動來維持投入，卡關用提示幫忙，而不是給跳過鈕。

## 7. 舉例／教材內容要「貼近小朋友生活」且「資訊完整」

**這次的問題**：教學的示範句一開始舉的例子（皇冠幾點不見）離小朋友太遠；而且「太長」的示範句其實是話講一半、資訊不完整——那問題就不只是「太長」了。

**建議 spec 這樣寫**：示範例子用**小朋友日常會遇到的情境**（借文具、跟老師請假、點餐、約碰面…）；示範「太長」時，**資訊要完整、只是囉唆繞圈**，這樣教的點才純粹。

---

## 一句話總結

> 未來的 spec 若能做到：**每段文字都配一個互動、概念寫成看得見的玩法、謎題附上推理鏈且不自相矛盾、角色先鋪陳再使用、資訊先給再讓玩家推理**，開發時就能一次到位，不用來回大改。
